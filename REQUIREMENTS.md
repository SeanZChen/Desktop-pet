# macOS 桌面宠物 - 需求文档与开发计划

## 一、需求概述

### 项目名称
DesktopPet（桌面宠物）

### 目标
开发一个运行在 macOS 桌面上的可爱宠物应用，宠物显示在桌面右下角，点击时会弹出打招呼对话框。

### 核心需求

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| R1 | 宠物显示在桌面右下角位置 | 高 |
| R2 | 宠物有持续的动画效果（如呼吸、摇摆） | 高 |
| R3 | 点击宠物弹出随机打招呼对话框 | 高 |
| R4 | 对话框3秒后自动消失 | 高 |
| R5 | 支持关闭宠物（右键菜单） | 中 |
| R6 | 宠物窗口始终在最顶层 | 高 |
| R7 | 支持多桌面切换跟随 | 中 |

### 扩展需求（可选）

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| E1 | 多种宠物形象切换 | 低 |
| E2 | 宠物状态动画（睡觉、玩耍） | 低 |
| E3 | 记住用户名字进行个性化问候 | 低 |
| E4 | 拖拽移动宠物位置 | 低 |

## 二、技术方案

### 技术选型

| 分类 | 技术 | 版本 | 理由 |
|------|------|------|------|
| 语言 | Swift | 5.9+ | macOS 原生开发首选语言 |
| UI框架 | SwiftUI | 5.0+ | 现代声明式UI框架 |
| 窗口类型 | NSWindow | - | 透明无边框窗口 |

### 架构设计

```
┌─────────────────────────────────────────────┐
│               AppDelegate                   │
│  - 管理宠物窗口生命周期                      │
│  - 处理屏幕变化事件                          │
│  - 定位窗口到右下角                          │
└───────────────┬─────────────────────────────┘
                │
                ▼
┌─────────────────────────────────────────────┐
│                PetView                      │
│  - 宠物形象渲染                             │
│  - 点击事件处理                             │
│  - 动画效果（摇摆）                          │
│  - 对话框显示/隐藏                          │
└─────────────────────────────────────────────┘
```

### 关键技术点

1. **透明无边框窗口**
   - 设置 `styleMask` 为 `.borderless`
   - 设置 `isOpaque = false`
   - 设置 `backgroundColor = .clear`

2. **右下角定位**
   - 获取主屏幕可见区域
   - 计算窗口位置：屏幕宽度 - 窗口宽度 - 边距

3. **窗口层级**
   - 设置 `level = .floating`（始终在最顶层）
   - 设置 `collectionBehavior` 支持多桌面

4. **动画效果**
   - 使用 SwiftUI 动画实现呼吸/摇摆效果
   - 使用 `repeatForever` 实现持续动画

5. **右键菜单**
   - 添加 NSMenu 实现关闭选项
